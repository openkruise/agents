This version supports the sandbox protocols as follows:
- E2B

For E2B Users:
1. Use {{ .Values.e2b.domain }} as E2B_DOMAIN in your agents
2. Make sure DNS record *.{{ .Values.e2b.domain }} is resolved to your ingress endpoint(s), you can view them with command:
    ```
    kubectl get ingress {{ include "acs-sandbox-manager.fullname" . }} -o jsonpath='{range .status.loadBalancer.ingress[*]}{.hostname}{.ip}{"\n"}{end}'
    ```
3. For HTTPS is required in E2B sdk, please issue a TLS certificate for the wildcard domain *.{{ .Values.e2b.domain }} and add it to the cluster using the following command:
    ```
    kubectl create secret tls {{ .Values.ingress.certSecretName }} \
          --cert=your-cert.pem \ # your cert file
          --key=your-key.pem \    # your key file
          --namespace={{ .Release.Namespace }} # istio-system if you use istio-gateway
    ```
